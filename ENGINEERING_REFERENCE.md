## 1. Software Architecture

DEAGUI is a **Qt 6 / C++20** single-window application with sidebar navigation. All application logic resides in the `MainWindow` class; there is no separate MVC framework.

```
main.cpp
  └── QApplication
        └── MainWindow : QMainWindow
              ├── QListWidget *sidebar          ← navigation
              ├── QScrollArea
              │     └── QStackedWidget *contentArea  ← 13 pages
              └── [per-page member widgets]
```

**Pages (`contentArea` index):**

| Index | Sidebar item | Creation method |
|---|---|---|
| 0 | 🏠 Home | `createHomePage()` |
| 1 | 🔄 File Conversion | `createFileConversionPage()` |
| 2 | 📊 Climate Visualisation | `createWeatherVisualizationPage()` |
| 3 | 🗺️ GeoJSON Visualisation | `createGeoJSONVisualizationPage()` |
| 4 | 🏗️ Component Editor | `createComponentsEditorPage()` |
| 5 | 📅 Schedule Management | `createSchedulesEditorPage()` |
| 6 | 📋 Building Types Table | `createBuildingTypesEditorPage()` |
| 7 | 🏢 Buildings Viewer | `createBuildingsEditorPage()` |
| 8 | ▶️ Run Simulation | `createSimulationPage()` |
| 9 | 📈 Simulation Results | `createSimulationResultsPage()` |
| 10 | ⚙️ Settings | `createSettingsPage()` |
| 11 | ℹ️ Info | `createInfoPage()` |
| 12 | 📞 Contact | `createContactPage()` |

Page switching is handled by slot `onMenuItemClicked()`:

```cpp
void MainWindow::onMenuItemClicked() {
    contentArea->setCurrentIndex(sidebar->currentRow());
}
```

## 2. Build System

**File**: `CMakeLists.txt`

```cmake
cmake_minimum_required(VERSION 4.1)
project(DEAGUI)

set(CMAKE_CXX_STANDARD 20)
set(CMAKE_AUTOMOC ON)   # automatic moc generation
set(CMAKE_AUTORCC ON)
set(CMAKE_AUTOUIC ON)

find_package(Qt6 COMPONENTS Core Gui Widgets Charts REQUIRED)
```

The `Q_OBJECT` macro is used in classes defined directly in `mainwindow.cpp` (`ToggleSwitch`, `ComponentsDelegate`). Because CMake AUTOMOC does not process non-header classes by default, the following line appears at the end of `mainwindow.cpp`:

```cpp
#include "mainwindow.moc"
```

**macOS AGL workaround**: the `AGL` framework is filtered out of the dependency list to avoid linker errors on modern macOS.

**Windows deployment**: `windeployqt.exe` is invoked automatically after the build to collect the required Qt DLLs into the output directory.

## 3. MainWindow Class

### 3.1 Constructor

```cpp
MainWindow::MainWindow(QWidget *parent)
    : QMainWindow(parent), totalDataPoints(0)
{
    setWindowTitle("Urban District Energy Analysis");
    resize(1400, 800);
    setMinimumSize(900, 600);
    setupUI();
    applyStyling();
    createPages();
}
```

Initialisation sequence: `setupUI()` → `applyStyling()` → `createPages()`.

### 3.2 setupUI()

Creates the main `QHBoxLayout` containing `sidebar` (QListWidget, width 200–250 px) and `contentArea` (QStackedWidget) wrapped in a `QScrollArea` with both horizontal and vertical scroll bars.

### 3.3 File I/O slots

A dedicated slot is defined for each I/O operation. The common pattern is:

1. `QFileDialog::getOpenFileName()` / `getSaveFileName()` for path selection.
2. Processing (reading, parsing, or calling an external binary).
3. Updating the display widgets.

### 3.4 External process execution

Binaries in the `bin/` folder are resolved by:

```cpp
QString MainWindow::getBinaryPath(const QString &binaryName)
```

which first searches the application executable's folder, then common system paths. Execution uses `QProcess`; stdout/stderr are forwarded to a `QTextEdit` console widget via the `readyReadStandardOutput` signal.

## 4. Input File Formats

### 4.1 WTST climate file

Whitespace- or tab-separated values with a header row. Typically contains: `timestamp`, `temperature`, `relative_humidity`, global/diffuse/direct solar radiation columns.

Generated by the `epw2wtst` converter from EnergyPlus EPW files.

### 4.2 Components JSON file

JSON document whose top-level keys correspond to building component categories (opaque walls, transparent surfaces, thermal bridges, HVAC). Each category is an array of objects with keys matching BldSimu simulator fields.

### 4.3 Building types CSV file

Generic CSV with a header row. Handled as a flat table by `QTableWidget`. No fixed schema is required.

### 4.4 Buildings JSON file

JSON document with an array of building objects. Each building includes:
- `SITE:LOCATION` — latitude, longitude, elevation, time zone.
- `BuildingSurface` array — name, construction type, adjacent zone.
- `Zone` array — name, area, volume, multiplier.

### 4.5 Simulation CSV file (BldSimu output)

Multi-zone format with a header row. Separator: comma.

**Global columns (required):**

| Column | Type | Unit | Source |
|---|---|---|---|
| `timestamp` | string/integer | — | Hour index |
| `T_out` | float | °C | `weatherrow_ptr->at(col_temperature)` |
| `x_ear` | float | kg_v/kg_as | `weatherrow_ptr->at(col_relativehumidity+1)` |

**Per-zone columns** (prefix `<zone_name>_`):

| Suffix | Type | Unit | Source |
|---|---|---|---|
| `_T_op` | float | °C | `thetaintop_hist` |
| `_T_air` | float | °C | `thataint` |
| `_x_iar` | float | kg_v/kg_as | `xint_hist` |
| `_Phi_C` | float | kWh/h | `PhiHC_hist` if positive (cooling load) |
| `_Phi_H` | float | kWh/h | `PhiHC_hist` if negative (heating load) |
| `_T_surf` | float | °C | Mean external surface temperature of the zone envelope (outer node of perimeter elements) |
| `_T_surf_dev` | float | °C | Standard deviation of external surface temperatures |

Example header for two zones:

```
timestamp,T_out,x_ear,Zone1_T_op,Zone1_T_air,Zone1_x_iar,Zone1_Phi_C,Zone1_Phi_H,Zone1_T_surf,Zone1_T_surf_dev,Zone2_T_op,...
```

`loadCSV()` detects zone names automatically from column prefixes. The legacy single-zone format (bare field names `T_op`, `T_air`, … without prefix) is also supported for backward compatibility.

### 4.6 Occupancy CSV file

```
timestamp,occupied
0,0
1,0
8,1
...
```

The occupancy vector length must equal the number of hours in the main dataset.

## 5. resilience_indicators.h Library

**Header-only C++17/20** library. No separate compilation is required.

### 5.1 Data structures

#### `ZoneData`

Holds data for a single thermal zone at one time step.

```cpp
struct ZoneData {
    // CSV inputs
    double T_op, T_air, x_iar, Phi_C, Phi_H, T_surf, T_surf_dev;
    // Derived fields (filled by computeDerivedFields)
    double SET, NG_kWh, EC_kWh, CO2_kg;
};
```

#### `HourlyData`

A complete hourly record for the whole building.

```cpp
struct HourlyData {
    double T_out, x_ear;            // Outdoor conditions
    std::vector<ZoneData> zones;    // One entry per zone
    bool occupied;                  // Occupancy flag
    // Aggregated fields (zone averages / sums)
    double T_dew, T_op, T_air, T_surf, x_iar, Phi_C, Phi_H,
           SET, NG_kWh, EC_kWh, CO2_kg;
};
```

#### `ResilienceParams`

All configurable parameters. Defaults are designed for an average Italian residential building:

```cpp
struct ResilienceParams {
    double T_comfort   = 26.0;    // [°C]
    double T_SET_alert = 28.0;    // [°C]
    double T_cold_safe = 17.78;   // [°C]
    double T_base      = 26.0;    // [°C]
    double floor_area  = 1.0;     // [m²]  — set to actual value
    double elec_conv   = 2.36;    // [-]
    double gas_conv    = 1.0;     // [-]
    double dt_h        = 1.0;     // [h]
    HeatingSystem heating_system = HeatingSystem::GAS_BOILER;
    double eta_H  = 0.85;
    double COP_C  = 3.0;
    double ef_gas  = 0.202;   // [kg_CO₂/kWh]
    double ef_elec = 0.233;   // [kg_CO₂/kWh]
    double met   = 1.2;   // [met]
    double clo   = 0.5;   // [clo]
    double v_air = 0.1;   // [m/s]
    double P_atm = 101325.0;  // [Pa]
};
```

### 5.2 Usage workflow

```cpp
// 1. Load data
auto data = loadCSV("simulation_output.csv");

// 2. (Optional) Apply occupancy
auto occ = loadOccupancy("occupancy.csv");
applyOccupancy(data, occ);

// 3. Compute derived fields
ResilienceParams params;
params.floor_area = 200.0;
computeDerivedFields(data, params);

// 4. Compute indicators — single dataset
SingleDatasetResults r1 = computeAllIndicators(data, params);

// 5. (Optional) Comparative indicators
auto data_ref = loadCSV("reference.csv");
applyOccupancy(data_ref, occ);
computeDerivedFields(data_ref, params);
TwoDatasetResults r2 = computeTwoDatasetIndicators(data_ref, data, params);
```

### 5.3 Thermodynamic helper functions

#### `satVaporPressure(T)` → [Pa]

Magnus formula:
```
Psat = 610.94 · exp( 17.625 · T / (T + 243.04) )
```

#### `dewPointFromHumRatio(x_ear, P_atm)` → T_dew [°C]

Magnus inverse (Alduchov & Eskridge 1996):
```
Pw    = x_ear · P_atm / (0.622 + x_ear)
γ     = ln(Pw / 610.94)
T_dew = 243.04 · γ / (17.625 − γ)
```

#### `computeSET(ta, tr, vel, x_iar, met, clo, P_atm)` → SET [°C]

Gagge 2-node model (ASHRAE 55-2020, Appendix D). Algorithm:

1. Compute skin temperature `Tsk` from the core-node energy balance at regulated core temperature (37 °C) with minimum skin conductance K_cr = 5.28 W/m²K.
2. Iteratively converge clothing surface temperature `Tcl`.
3. Compute required skin wettedness `w`.
4. Bisection to find SET: temperature of a uniform reference environment (ta = tr, v = 0.1 m/s, φ = 50 %) producing the same Tsk.

Mean radiant temperature estimate per zone:
```
T_mrt = 2 · T_op − T_air
```

#### `computeHumidex(T_air_C, T_dew_C)` → H [-]

```
E = 6.105 · exp( 25.22 · (Tdew_K − 273.16)/Tdew_K − 5.31 · ln(Tdew_K/273.16) )
H = T_air + 0.5555 · (E − 10)
```

### 5.4 Indicators — equations and implementation

#### In_I-B-1_07 — Unmet hours for Humidex

`indicatorHumidex(data)` → `HumidexResult`

For each occupied hour, H is computed and assigned to the Canadian thermal discomfort band:

| Band | H range |
|---|---|
| 0 | H < 27 |
| 1 | 27 ≤ H < 35 |
| 2 | 35 ≤ H < 40 |
| 3 | 40 ≤ H < 45 |
| 4 | 45 ≤ H < 54 |
| 5 | H ≥ 54 |

Output: `band_pct[6]` — percentage of occupied hours per band; `unmet_pct` — total of bands 1–5.

---

#### In_I-B-2_03 — Indoor Overheating Degree (IOD)

`indicatorIOD(data, T_comf, dt)` → [°C·h]

```
IOD = Σ_i max(0, T_op,i − T_comf) · Δt    (occupied hours only)
```

---

#### In_I-B-2_11 — Recovery time

`indicatorRecoveryTime(data, T_SET_alert, max_hours, dt)` → [h]

Locates the global SET peak. Then finds the first moment after the peak from which SET remains below `T_SET_alert` for at least `max_hours` (default 24) consecutive hours.

```
t_rec = t_below − t_peak   [h]
```

Returns −1 if SET never exceeds the threshold or never recovers.

---

#### In_I-B-2_13 — Absorptivity time

`indicatorAbsorptivityTime(data, T_SET_alert, dt)` → [h]

```
t_abs = t_peak − t_start   [h]
```

where `t_start` is the first hour SET exceeds `T_SET_alert` and `t_peak` is the global SET maximum.

---

#### In_I-B-2_08 — Overheating Escalation Factor (aIOD)

`indicatorOEF(iod_values, awd_values)` → [-]

Simple linear regression of IOD on AWD over N scenarios:

```
aIOD = Cov(IOD, AWD) / Var(AWD)
```

- aIOD < 1 → resilient building (mitigates external thermal stress)
- aIOD > 1 → vulnerable building (amplifies overheating)

Requires at least 2 (IOD, AWD) pairs from distinct scenarios.

---

#### In_O-B-1_01 — Ambient Warmness Degree (AWD)

`indicatorAWD(data, T_base, dt)` → [°C·h]

```
AWD = Σ_i max(0, T_out,i − T_b) · Δt    (occupied hours only)
```

---

#### In_O-B-2_02 — Reduction in annual CO₂ emissions

`indicatorCO2Reduction(data_ref, data_bldg)` → [kg/yr]

```
ΔCO₂ = Σ CO₂_ref,i − Σ CO₂_bldg,i
```

Positive value = emission reduction relative to the reference.

---

#### In_O-B-2_03 — Primary energy consumption

`indicatorPrimaryEnergy(data, params)` → [kWh/(m²·yr)]

```
PE = (Σ NG_i · f_gas + Σ EC_i · f_elec) / A_floor
```

`NG_i` and `EC_i` are computed by `computeDerivedFields()` based on the HVAC system type:

```
// Gas boiler
NG_kWh = Phi_H / eta_H
EC_kWh = Phi_C / COP_C

// Electric heat pump
NG_kWh = 0
EC_kWh = Phi_H / COP_H + Phi_C / COP_C
```

---

#### In_O-B-3_07 — Reduction in external surface temperature

`indicatorSurfTempReduction(data_ref, data_bldg)` → [°C]

```
ΔT_surf = mean(T_surf_ref) − mean(T_surf_bldg)
```

---

#### In_I-A-1_01 — Seasonal Building Overheating Index (SBOI)

`indicatorSBOI(data, T_base)` → [%]

```
SBOI = [ Σ max(0, T_op,i − T_out,i) ] / [ Σ max(0, T_out,i − T_b) ] × 100
```

- SBOI < 100 % → building mitigates external overheating
- SBOI > 100 % → building worsens indoor conditions

---

#### In_I-A-2_01 — Resilience Class Index (RCI)

`indicatorRCI(data_ref, data_bldg, params)` → [-]

Intermediate: Weighted Unmet Thermal Performance (winter scenario — heating interruption):

```
WUMTP = Σ max(0, T_cold_safe − T_op,i) · Δt   [°C·h]
RCI   = WUMTP_ref / WUMTP_bldg
```

RCI > 1 → greater resilience than the reference.

---

#### In_I-A-2_02 — Thermal Resilience Index (TRI)

`indicatorTRI(data_orig, data_retro, params)` → [-]

Intermediate: Whole-building SET-Hours above threshold (summer scenario — heat wave):

```
WSETH = Σ max(0, SET_i − SET_alert) · Δt   [°C·h]
TRI   = WSETH_original / WSETH_retrofitted
```

TRI > 1 → the retrofit improved thermal resilience.

### 5.5 Aggregation functions

```cpp
// All single-dataset indicators
SingleDatasetResults computeAllIndicators(
    const std::vector<HourlyData>& data,
    const ResilienceParams& params = {});

// All comparative indicators (two datasets)
TwoDatasetResults computeTwoDatasetIndicators(
    const std::vector<HourlyData>& data_ref,
    const std::vector<HourlyData>& data_bldg,
    const ResilienceParams& params = {},
    const std::vector<double>& iod_scenarios = {},
    const std::vector<double>& awd_scenarios = {});
```

`SingleDatasetResults` fields: `humidex`, `IOD`, `recovery_time_h`, `absorptivity_time_h`, `AWD`, `primary_energy`, `SBOI`.

`TwoDatasetResults` fields: `CO2_reduction`, `surf_temp_reduction`, `RCI`, `TRI`, `OEF`.

## 6. External Tools

Binaries reside in the `bin/` folder relative to the application executable. Path resolution is performed by `getBinaryPath()`.

| Binary | Invocation | Output |
|---|---|---|
| `epw2wtst` | `epw2wtst <input.epw> <output.wtst>` | WTST file |
| `idf2json` | `idf2json <input.idf> <output.json>` | Components JSON file |
| `shaperead` | `shaperead <input.shp> <output.geojson>` | GeoJSON file |
| `pol_geoelem` | `pol_geoelem <input.geojson> <output.geojson>` | Processed GeoJSON file |
| `BldSimu` | `BldSimu --mode <iso\|noise> <climate> <comp> <sched> <bldg>` | Simulation output CSV |

Processes are started with `QProcess`; stdout and stderr are forwarded to the `simulationOutputConsole` (QTextEdit) via the `readyReadStandardOutput` signal.

## 7. Custom Widgets

### ComponentsDelegate

`QStyledItemDelegate` for the components table. For cells corresponding to fields with known value sets (e.g. `roughness`, `gas_type`, `thermostat_type`) it presents an editable `QComboBox` with static suggestions. All other fields use the default editor.

```cpp
static const QMap<QString, QStringList> &staticSuggestions()
```

### ToggleSwitch

`QAbstractButton` with an iOS-style appearance. The animated `offset` property (0.0 → 1.0) is driven by `QPropertyAnimation` with a 150 ms duration and `InOutQuad` easing curve. Custom `paintEvent()` draws a rounded track and a circular knob coloured green (#27ae60) when active.
